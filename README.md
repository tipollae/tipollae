### My journey
One of my earlier projects, Yuzzwatch, was a real-time watch together platform built with Node.js, Socket.IO, MongoDB, and the YouTube API. The project worked, but as it grew I began noticing issues with security, scalability, project structure, and maintainability.

Rather than viewing those problems as failures, I treated them as opportunities to improve.

Since then, every project I've worked on has been an attempt to solve a weakness I discovered in a previous one:
- Improving authentication and session handling.
- Designing cleaner backend architectures.
- Building more maintainable database structures.
- Thinking about scalability before adding features.
- Writing code that is easier to extend and refactor.
- Understanding how real-world systems handle large numbers of users.

I don't aim to write perfect software on the first attempt.
Instead, I focus on identifying weaknesses, understanding why they exist, and applying those lessons onto the next thing that I work on.
